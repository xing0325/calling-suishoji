# CALLING 移动端交互优化指南

## 核心问题分析

### 当前问题
1. **滑动误触** - 滑动阈值太低，容易在滚动时误触发页面切换
2. **卡片过长** - 待办卡片在手机端占用过多空间，内容溢出
3. **动画生硬** - 页面切换动画不够流畅优雅

## 最佳实践总结

### 1. 手势设计原则
- **避免过度使用手势**：不要让滑动同时用于多个目的（如既用于页面切换，又用于列表操作）
- **提供清晰的反馈**：用户执行手势时要有即时的视觉反馈
- **支持标准手势**：优先使用点击（tap），而不是复杂的滑动
- **降低误触风险**：增加滑动阈值，避免滚动时误触

### 2. 滑动交互改进
- **增加滑动距离阈值**：从50px提高到80-100px，减少误触
- **添加滑动方向检测**：只在明确的水平滑动时触发页面切换
- **提供视觉指示**：显示当前页面指示器（dots或tabs）
- **支持取消操作**：滑动过程中可以通过反向滑动取消

### 3. 动画优化
- **使用缓动函数**：cubic-bezier(0.4, 0, 0.2, 1) 或 ease-out-cubic
- **动画时长**：300-400ms为最佳（不要太快或太慢）
- **响应式动画**：根据滑动速度调整动画时长
- **避免闪烁**：使用 transform 和 opacity 而不是 left/top

### 4. 手机端卡片优化
- **紧凑设计**：卡片高度控制在100-120px以内
- **信息优先级**：只显示最重要的信息，详情通过点击展开
- **缩小间距**：减少卡片间的margin，从16px改为8-12px
- **简化操作**：按钮改为图标，减少视觉复杂度

### 5. 页面导航设计
- **底部导航固定**：便于单手操作
- **当前页面高亮**：使用颜色和缩放效果区分
- **标签栏指示器**：显示当前页面位置

## 实现策略

### 滑动切换优化
```
1. 增加滑动阈值到80px
2. 检测滑动方向（水平vs垂直）
3. 垂直滑动距离>20px时，忽略水平滑动
4. 添加页面指示器（dots）
5. 支持通过底部导航快速切换
```

### 卡片优化
```
1. 卡片高度：100px（展开状态）
2. 标题+进度条+摘要（3行以内）
3. 点击展开查看详细内容
4. 操作按钮改为右侧图标
```

### 动画优化
```
1. 页面切换：300ms cubic-bezier(0.4, 0, 0.2, 1)
2. 按钮点击：150ms ease-out（缩放0.95）
3. 卡片展开：200ms ease-out
4. 使用 will-change 优化性能
```
