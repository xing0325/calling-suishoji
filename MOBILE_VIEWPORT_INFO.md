# 手机屏幕适配指南

## 主流手机屏幕分辨率（2026年）

### 全球主流分辨率排名
1. **414x896** - 12.43% (iPhone 11, 12, 13, 14, 15)
2. **360x800** - 9.58% (Android 标准)
3. **390x844** - 6.81% (Pixel 6/7/8)
4. **384x832** - 4.42% (Pixel 5/5a)
5. **393x873** - 4.15% (Pixel 8 Pro)
6. **360x780** - 3.49% (Android 标准)

## 视口宽度（Viewport Width）

### iOS
- iPhone 6-15: **375px** (逻辑像素)
- iPhone 6+ 及以上: **414px** (逻辑像素)
- iPhone 12 Pro Max: **428px** (逻辑像素)

### Android
- 标准基准: **360px** (逻辑像素)
- 常见: 360px, 375px, 390px, 412px

## 设计建议

### 1. 响应式设计断点
```
- 手机: 320px - 480px (最小)
- 手机: 480px - 768px (标准)
- 平板: 768px 以上
```

### 2. 安全区域
- 最小宽度: **320px** (兼容旧设备)
- 推荐宽度: **360px** (Android 标准)
- 优化宽度: **375px** (iOS 标准)
- 最大宽度: **430px** (大屏手机)

### 3. 单页面宽度
- **不应该** 让单个页面超过视口宽度
- 每个页面应该 **恰好占满视口宽度** (100vw 或 100%)
- 这样滑动时能看到完整的一个页面

## CALLING 应用适配方案

### 当前问题
- 每个页面太宽，无法在屏幕上呈现完整的一个页面
- 滑动时看不到真正的页面内容，停不住

### 解决方案
1. 设置 `max-width: 100vw` 或 `width: 100%`
2. 每个页面宽度 = 视口宽度
3. 5个页面总宽度 = 500vw
4. 滑动时准确停在有内容的页面

### 实现细节
```css
/* 容器 */
.page-container {
  width: 100%;
  overflow: hidden;
}

/* 页面包装器 */
.pages-wrapper {
  display: flex;
  width: 500%; /* 5个页面 */
}

/* 单个页面 */
.page {
  width: 20%; /* 1/5 */
  flex-shrink: 0;
}
```

## 适配步骤
1. ✅ 设置正确的 viewport meta 标签
2. ✅ 使用 100% 宽度而不是固定像素
3. ✅ 确保每个页面恰好占满视口
4. ✅ 调整滑动阈值为视口宽度的 30-50%
